タスクディレクトリに置かれた `plan.md`（grill-me で人間が確定した正本）の妥当性を検証し、`plan-validation` 出力契約に従ってレポートを作成してください。**新しいプランは作らない。**

## 読み込み（必須・最初に実行）

1. タスク指示書（`order.md`）を読み、目的・受け入れ基準・参照資料を把握する
2. `order.md` の参照資料に従い、タスクディレクトリ内の `plan.md` を Read で開く
3. `plan.md` が存在しない、または参照パスが解決できない場合は検証不能として報告する

## 検証観点

| 観点 | 確認内容 |
|------|----------|
| 完全性 | 目的・分解した要件・スコープ・実装アプローチが揃っているか |
| 実装可能性 | 現行コードベース上で実行可能か。Read / Glob / Grep で調査する |
| 一貫性 | `order.md` の要求と `plan.md` が矛盾していないか |
| スコープ | 要求外の作業が混入していないか。不足している明示要求がないか |
| 検証可能性 | 各要件が実装後に確認できる粒度まで分解されているか |
| 参照の実在 | `plan.md` や `order.md` が指すファイル・ディレクトリが存在するか |

## 判定

レポートの結果は **APPROVE** または **REJECT** のいずれかとする。

- **APPROVE**: 致命的な不明点・矛盾がなく、実装に進める
- **REJECT**: 実装前に修正が必要な問題がある。曖昧さや確認事項も blocking 問題として列挙する

## 禁止事項

- `plan.md` の内容を書き換えたり、新規プランを生成したりしない
- 参照資料を開かずに推測で妥当性を判定しない
- 一般的ベストプラクティスを理由に、要求外の作業を必須化しない
