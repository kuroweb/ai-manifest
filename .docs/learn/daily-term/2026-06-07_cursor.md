# 2026-06-07_cursor

## source_session が issue slug として使われていた

- **失敗・手戻り:** `source_session` を「会話識別子」と定義しながら、実サンプルでは `knowledge-accumulation-workflow`（issue slug）が入っていた。issue ワークフローへの暗黙依存になっていた。
- **原因:** `/issue` 起点の作業では作業文脈が issue slug になりやすく、自動取得できない session ID の代わりにそれを流用してしまった。
- **根拠:** ユーザーから「source_session の意味合いが実質的に issue.md に依存」「source_issue 不要、source_session 自体不要」と指摘。調査の結果、3 エージェントともプロンプトへ session ID は渡されないことも確認した。
- **次回の対策:** frontmatter に自動取得できない識別子は入れない。`agent` + `created_at` + ファイル名で足りる。逆引きは incident の `根拠` に書く。

## HHMM 削除後も _2/_3 衝突ルールが残っていた

- **失敗・手戻り:** ファイル名から `HHMM` を外したあとも「同名衝突時は `_2` `_3`」が残り、生成ルールの「既存ファイルを更新」と矛盾していた。
- **原因:** `HHMM` あり時代の複数セッション分離ルールを、ファイル名変更時に衝突方針まで見直さなかった。
- **根拠:** ユーザーから「衝突時は更新じゃなかったっけ？」と指摘された。
- **次回の対策:** ファイル名規則を変えたら、衝突時の挙動（新規 vs 更新）もセットで見直す。`YYYY-MM-DD_<agent>.md` では同名＝更新に統一する。

## 合意後の実装を質問で遅らせた

- **失敗・手戻り:** テンプレート命名の案 A やスコープ最小など合意済みの内容について、実装前に追加確認を重ね、ユーザーから「はよ修正しろ」と指摘された。
- **原因:** issue ワークフローの grill-me を、方針が固まった後も機械的に継続した。
- **根拠:** ユーザーが「推奨回答」「A」「推奨対応」と短く承認しているのに、次の質問や確認を挟んだ。
- **次回の対策:** 推奨案への明確な承認が出たら、残る未確定論点がなければ質問を止めて実装に移る。

## テンプレート frontmatter の命名がディレクトリとずれていた

- **失敗・手戻り:** `source_long` `source_short` など、テンプレートの frontmatter キーが `long-term` `short-term` ディレクトリ名と揃っておらず、ユーザーが個別に修正を指示する必要があった。
- **原因:** 初版テンプレート作成時に、データディレクトリ名との対応を frontmatter キーまで一貫して揃えなかった。
- **根拠:** ユーザーから `source_long` → `source_long_term`、`source_short` → `source_short_term` の変更指示があった（命名揃え issue の続き）。
- **次回の対策:** `learn/**` のディレクトリ名を決めたら、テンプレートの frontmatter キーも `source_<dir-name>` 形式で同時に揃える。
