# rulesync-overview-refinement

- **プロジェクト名:** ai-manifest
- **作成日:** 2026-05-05

## 目的

`.rulesync/rules/overview.md` に、エージェントが迷わない最小限の常時ルールを追加する。  
特に「正本は `.rulesync/`、生成物は直接編集しない」という境界を明確にする。

## 当面のスコープ

今回は `overview.md` に `## rulesync（正本と生成物）` を追加するところまで。  
他の論点や見出し拡張は後続で扱う。

## 編集方針（overview.md）

- 毎セッションで読む前提の短文ルールに限定する。
- 長文説明や計画メモは書かない。断定・命令形で書く。
- 用途限定の手順（メモ作成や引き継ぎ運用など）は `SKILL.md` 側に置き、`overview.md` に複製しない。
- 詳細説明、表、チェックリストは `README.md` または `.rulesync/rules/<topic>.md` に置く。

## ベストプラクティス観点（CLAUDE.md / AGENTS.md / GEMINI.md）

各エージェントの公式ドキュメントを踏まえ、`overview.md` には次の原則を反映する。

- **短く具体的に書く**  
  すべての instruction ファイルは毎セッション読み込まれるため、長文化すると遵守率が下がる。  
  抽象語より、実行可能な短文ルールを優先する。
- **毎回必要な情報だけを置く**  
  常時必要なルール（編集境界、必須コマンド、禁止事項）のみに限定する。  
  条件付き手順や長い背景説明は別ファイルに分離する。
- **階層ロード前提で、重複を最小化する**  
  CLAUDE / Codex / Gemini いずれも階層的に instruction を読み込む。  
  上位ファイルは共通原則、下位ファイルは局所ルールに寄せる。
- **「何をしないか」を明記する**  
  実施事項だけでなく、禁止事項（生成物の直接編集禁止など）を明文化する。
- **運用で更新する前提にする**  
  同じ誤りが繰り返されたら instruction を更新する。  
  1回で完璧を目指さず、短く保って継続改善する。

## 今回の issue への反映ポイント

- `overview.md` は「正本と生成物の境界」を最優先の常時ルールとして明記する。
- それ以外の運用詳細（背景、手順、例外処理）は `README.md` や各 `SKILL.md` に置く。
- 将来の拡張時も、`overview.md` 自体は短く保ち、トピックは別ファイルに逃がす。

## 追加する本文の要点

- `.rulesync/` を正本として扱う。
  - ルール・スキル・サブエージェントの変更は `.rulesync/` のみ編集する。
- `rulesync generate` の出力物（`AGENTS.md` / `CLAUDE.md` / `GEMINI.md`、各エージェント向け rules/memories など）は直接編集しない。
  - 内容を変える場合は `.rulesync/` を編集してから `rulesync generate` を実行する。

## 体裁ルール

- `overview.md` のフロントマターは現状維持（`root: true`）。
- 見出しは深くしない（`###` 以降は原則使わない）。
- 見出し構成は以下のいずれか:
  - `# Overview` + 箇条書きのみ
  - `# Overview` + `## rulesync（正本と生成物）`

## ドキュメント分担

| 置き場 | 役割 |
| --- | --- |
| `README.md` | 人間向けの恒久手順・仕様（セットアップ、コマンド、禁止事項など） |
| `.rulesync/rules/overview.md` | エージェント向けの常時ルール（編集境界を短文で定義） |
| `.rulesync/rules/*.md` | トピック別ルール（命名、スキル選択など） |
| `.rulesync/skills/*/SKILL.md` | 用途限定ワークフロー（いつ何をするか） |
| `rulesync generate` の出力物 | 正本ではないため直接編集しない |

## 完了条件

- `.rulesync/rules/overview.md` に `rulesync` 境界の H2 と箇条書きが追加されている。
- 追加文が短文・命令形で、手順メモを含まない。
- `.rulesync/` 編集後に `rulesync generate` を実行し、生成物へ反映できる状態になっている。
