# Issue ワークフロー

思いつき・改善・試したいことを、相談から着手まで一貫して進める入口スキル。
正本は `~/.docs/issues/<state>/<slug>/issue.md` のみとする。

## トリガー

- `/issue`
- 「やりたいこと相談」「issue を作成」「issue 再開」「前回の issue の続き」
- 「issues にメモして」「調査用の issue を作って」

## 起動時の分岐

`/issue` 起動後、次のいずれかをユーザーに確認する。

1. **新規** — やりたいことを一言で受け取り、draft issue を即作成してから検討へ
2. **既存を開く** — `draft` / `ready` の issue から選んで読み込む
   - `draft` なら検討フェーズを継続
   - `ready` なら実装フェーズへ

候補が複数あるときは勝手に選ばず、番号または slug で指定してもらう。

### 既存 issue 選択テンプレート

```text
issue を開きます。どれを選びますか？
1. draft/<slug1>/issue.md
2. ready/<slug2>/issue.md
番号、slug、またはファイルパスで指定してください。
```

## 実行フロー

### フェーズ 1: 新規 issue 作成

1. やりたいことを一言で受け取る
2. slug を自動生成する（小文字・数字・ハイフンのみ）
3. `~/.docs/issues/draft/<slug>/issue.md` を **検討に入る前に** 作成する
4. `assets/issue.md` をベースに、概要に一言メモを反映する
5. 作成日を `YYYY-MM-DD` で記入する
6. プロジェクト名を補完する（未確定なら `unknown`。わかっていれば作成直後に埋める）
7. ファイルの存在と必須項目を確認する

### フェーズ 2: 検討（grill-me）

1. `grill-me` スキルの手順に従う
2. 質問は一度に一つ。各質問に推奨回答を示す
3. コードベースで答えられることはユーザーに聞かず調べる
4. 合意が得られた内容は、その都度 `issue.md` の「実施計画」「次のアクション」に追記する
5. ユーザーが `grill-with-docs` を明示したときだけ、`grill-with-docs` に切り替える

### フェーズ 3: ready 昇格

次をすべて満たし、ユーザーが「着手できる」「ready にして」と言ったときだけ昇格する。

- 「概要」が記入済み
- 「実施計画」が記入済み
- 「次のアクション」が記入済み

ユーザーが ready 昇格を指示したら、対象 issue ディレクトリを `~/.docs/issues/ready/<slug>/` へ移動する。
`issue.md` 以外の添付も一緒に移動する。

### フェーズ 4: 実装

- `ready` の issue を読み、次のアクションから着手する
- 進捗に応じて `issue.md` を更新する
- タスクが完了したらチェックを付ける

### フェーズ 5: close

- 対応完了または対応不要と判断したら、ユーザーが「close して」「終了して」と言ったときに実行する
- 対象 issue ディレクトリを `~/.docs/issues/close/<slug>/` へ移動する
- `issue.md` 以外の添付も一緒に移動する

### draft への差し戻し

ユーザーが「draft に戻して」と言ったとき、対象 issue ディレクトリを `~/.docs/issues/draft/<slug>/` へ移動する。
`issue.md` 以外の添付も一緒に移動する。

## 既存 issue を開いたとき

`issue.md` を読み込み、次の 4 点を簡潔に共有する。

- 概要
- 次のアクション
- 未完了タスク
- 注意事項（決定事項・詰まりやすい点）

issue 本文の長文転記は避ける。

- `draft` ならフェーズ 2（検討）を継続する
- `ready` ならフェーズ 4（実装）へ進む

## ルール

### 正本

- 記録の正本は `issue.md` のみ

### slug

- 使える文字は小文字・数字・ハイフンのみ
- `issue.md` の H1 に使う
- 同名 slug が既にある場合は別 slug を提案するかユーザーに確認する

### 状態管理

| 状態 | ディレクトリ | 意味 |
| --- | --- | --- |
| `draft` | `~/.docs/issues/draft/` | 検討中・整理途中 |
| `ready` | `~/.docs/issues/ready/` | 着手可能 |
| `close` | `~/.docs/issues/close/` | 完了または終了 |

### 必須セクション（新規 issue）

- プロジェクト名 / 作成日 / 概要 / 実施計画 / 次のアクション

### 任意セクション

- メモ / タスク

### バリデーション

- `~/.docs/issues/` がなければ作成する
- issue 作成後にファイルの存在と必須項目を確認する
- サンドボックス外への書き込みが必要な場合は権限昇格を求める
