# テストシナリオ

user story をもとに、テスト目的、開始条件、ユーザーロール、段階的な操作、期待結果を含む包括的なテストシナリオを作る。

**使いどころ:** QA テストケース作成、テスト計画作成、受け入れテストシナリオ定義、user story 実装の検証。

**引数:**

- `$PRODUCT`: プロダクト名またはシステム名
- `$USER_STORY`: テスト対象の user story (タイトルと受け入れ条件)
- `$CONTEXT`: 追加のテスト文脈や制約

## 進め方

1. **user story と受け入れ条件を確認する**
2. **テスト目的を定義する** - 何の振る舞いを確かめるか
3. **開始条件を整理する** - システム状態、データ、設定
4. **ユーザーロールを特定する** - 誰が操作するか
5. **テスト手順を作る** - 操作を段階的に分解する
6. **期待結果を定義する** - 各手順のあとに何が観測されるか
7. **エッジケースを考える** - 不正入力、境界値、例外
8. **詳細なテストシナリオを出す** - QA がそのまま使える形にする

## テンプレート

**Test Scenario:** [わかりやすいシナリオ名]

**Test Objective:** [このテストで何を検証するか]

**Starting Conditions:**

- [必要なシステム状態]
- [必要なデータや設定]
- [ユーザー権限や事前準備]

**User Role:** [テスト実行者]

**Test Steps:**

1. [最初の操作と期待される結果]
2. [次の操作と観測結果]
3. [さらに続く操作とシステム挙動]
4. [完了操作と最終状態]

**Expected Outcomes:**

- [観測結果 1]
- [観測結果 2]
- [観測結果 3]

## 例

**Test Scenario:** 商品詳細ページで最近見た商品を表示する

**Test Objective:** `Recently viewed` セクションが正しく表示され、現在の商品を含まないことを確認する。

**Starting Conditions:**

- ユーザーはログイン済み、またはブラウザ履歴が有効
- 現在の session で少なくとも 2 商品を閲覧済み
- 今見ている商品は過去に見た商品と別

**User Role:** Online Shopper

**Test Steps:**

1. 任意の商品詳細ページへ移動する → ページ下部に過去に見た商品が表示される
2. ページ下部までスクロールする → `Recently viewed` セクションと商品カードが見える
3. サムネイルを確認する → 画像、タイトル、価格が正しく表示される
4. 現在の商品を確認する → 現在の商品は一覧に含まれない
5. 商品カードをクリックする → 対応する商品詳細ページへ遷移する

**Expected Outcomes:**

- 1 つ以上の過去閲覧商品がある場合のみセクションが表示される
- 4-8 個の商品カードが必要情報つきで表示される
- 現在の商品は除外される
- 各カードに `Viewed X minutes/hours ago` の時刻表現が出る
- クリックで正しいページへ移動する
- 表示は 2 秒以内に完了する

## 成果物

- 各受け入れ条件に対応した包括的なテストシナリオ
- user story の意図に沿った明確なテスト目的
- 詳細な手順
- 各手順後に観測できる期待結果
- エラー系、境界条件、エッジケースのカバー
- QA チームがそのまま使える粒度のドキュメント
